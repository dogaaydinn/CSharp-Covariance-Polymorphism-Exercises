Issue 5: No DTOs

What I'm thinking:

"Junior is exposing domain models directly. This is going to cause so many problems: security (password field), over-posting (client can set Id), tight coupling (can't change domain without breaking API)."

Why DTOs matter (my explanation):

WITHOUT DTOs:
Client sends: { "id": 999, "name": "Hacker", "isAdmin": true }
‚Üí Over-posting! Client set their own ID and admin flag!

WITH DTOs:
Request DTO only has: name, email, password
‚Üí Client CAN'T set id, isAdmin, createdAt
‚Üí Server controls these fields

Over-posting attack I'll show junior:

// ‚ùå WITHOUT DTO:
[HttpPost]
public User CreateUser(User user) // ‚Üê Accepts entire User object!
{
    _users.Add(user); // ‚ùå Client could have set: Id=1, IsAdmin=true!
    return user;
}

// ‚úÖ WITH DTO:
[HttpPost]
public UserDto CreateUser(CreateUserRequest request) // ‚Üê Only accepts: name, email, password
{
    var user = new User
    {
        Id = _idGenerator.Next(), // ‚Üê Server controls ID
        Name = request.Name,
        Email = request.Email,
        IsAdmin = false, // ‚Üê Server controls admin flag
        CreatedAt = DateTime.UtcNow
    };
    // ...
}
